Type Definitions

AssetIdPathMap

A plain object that is a map of asset IDs to their respective web paths. Contains only one type of asset, whether stylesheets or images, etc.

Type: Object
Example
{
	"overlay": "/css/gray-overlay.css",
	"dialogBox": "/css/schnazzy-dialog.css"
}

AssetsObject

A plain object that could contain multiple AssetIdPathMaps. Each key in the top level of an AssetsObject must be a SupportedAssetType. Each gets its own property to show what type of assets it references.

When loading JavaScript, if more paths are added to the AssetsObject, the AssetLoader's done callback will not fire until those newly added assets are loaded too, and any new assets they add... and so on.

Type: Object
Properties:
Name Type Argument Description
css AssetIdPathMap <optional>

css assets to be loaded
js AssetIdPathMap <optional>

javascript assets to be loaded
img AssetIdPathMap <optional>

image assets to be loaded
hbs AssetIdPathMap <optional>

Handlebars templates to be loaded
Examples
var assetsObj = {
  css: {
    "foo": "/path/to/foo.css"
  },
  js: {
    "foo": "/path/to/foo.js",
    "bar": "/path/to/bar.js"
  }
};

Concatenated assets can have their constituent asset identifiers marked as loaded by AssetLoader.hasLoaded while pointing to only one path. Use the | pipe character to separate asset IDs in the key of an AssetIdPathMap.

var assetsObj = {
  css: {
    "foo|bar|baz": "/min/foobarbaz.min.css"
  }
};

LoaderCallbackSetter(callback)

Parameters:
Name Type Description
callback function

to be called when the given event occurs. Will be passed two String arguments: assetType, assetId. For error handlers, there could also be one or two additional parameters: path, errorMessage.
Returns: LoaderPromise

The calling object for method chaining, so you can set different callbacks in one line of code.

LoaderPromise

A promise object that allows for assigning callbacks to loading events. Each of its functions returns the LoaderPromise, so you can chain method calls.

Type: Object
Properties:
Name Type Description
done LoaderCallbackSetter

register a callback to be fired after the loader has loaded all desired assets.
each LoaderCallbackSetter

register a callback for the loader to fire after each asset it loads.
error LoaderCallbackSetter

register an error handler to be called when the loader encounters an error loading an asset.
Example
var promise = AssetLoader.load("/css/flowers.css", "/js/flowers-petals.js");
promise.each(singleAssetLoadedHandler).done(flowersLoadedCallback).error(errHandler);

SupportedAssetType

One of the following: 'css', 'hbs', 'img', 'js'

Type: String
